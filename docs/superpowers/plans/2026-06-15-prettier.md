# Prettier Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Prettier for code formatting to both the frontend and backend, integrated with ESLint and VS Code save-on-format.

**Architecture:** Single root `.prettierrc` covers both projects. `eslint-config-prettier` is added last in each ESLint config to disable any formatting rules that conflict with Prettier (including `no-multiple-empty-lines`, which Prettier handles itself). Format-on-save added to `.vscode/settings.json` alongside existing ESLint autofix.

**Tech Stack:** Prettier 3, eslint-config-prettier, VS Code extension `esbenp.prettier-vscode`

## Task 2: Add Prettier to the backend

**Files:**

- Modify: `backend/eslint.config.mjs`

- [ ] **Step 1: Install prettier and eslint-config-prettier in the backend**

```bash
cd backend && npm install --save-dev prettier eslint-config-prettier
```

Expected: packages added to `backend/package-lock.json`, no errors.

- [ ] **Step 2: Add format scripts to `backend/package.json`**

Add to the `"scripts"` section (after `lint:fix`):

```json
"format": "prettier --write .",
"format:check": "prettier --check ."
```

The full `scripts` block should look like:

```json
"scripts": {
  "start:dev": "nest start --watch",
  "build": "nest build",
  "test": "jest",
  "test:watch": "jest --watch",
  "typecheck": "tsc --noEmit",
  "lint": "eslint .",
  "lint:fix": "eslint . --fix",
  "format": "prettier --write .",
  "format:check": "prettier --check ."
}
```

- [ ] **Step 3: Update `backend/eslint.config.mjs` to disable conflicting rules**

Replace the entire file with:

```js
import tseslint from 'typescript-eslint';
import prettierConfig from 'eslint-config-prettier';

export default tseslint.config(
  { ignores: ['dist', 'node_modules'] },
  {
    files: ['**/*.ts'],
    extends: [tseslint.configs.recommended],
    rules: {
      '@typescript-eslint/no-explicit-any': 'warn',
    },
  },
  prettierConfig,
);
```

Note: `no-multiple-empty-lines` is removed from `rules` — `eslint-config-prettier` disables it because Prettier handles blank lines itself. `prettierConfig` is placed last so it overrides any formatting rules from previous configs.

- [ ] **Step 6: Verify lint still passes**

Expected: Same errors as before (pre-existing `no-unused-vars` in spec file, `no-explicit-any` warnings). No new errors introduced by the config change.

- [ ] **Step 3: Update `frontend/eslint.config.js` to disable conflicting rules**

Replace the entire file with:

```js
import js from '@eslint/js'
import globals from 'globals'
import reactHooks from 'eslint-plugin-react-hooks'
import reactRefresh from 'eslint-plugin-react-refresh'
import tseslint from 'typescript-eslint'
import eslintConfigPrettier from 'eslint-config-prettier'
import { defineConfig, globalIgnores } from 'eslint/config'

export default defineConfig([
  globalIgnores(['dist']),
  {
    files: ['**/*.{ts,tsx}'],
    extends: [
      js.configs.recommended,
      tseslint.configs.recommended,
      reactHooks.configs.flat.recommended,
      reactRefresh.configs.vite,
    ],
    languageOptions: {
      globals: globals.browser,
    },
  },
  eslintConfigPrettier,
])
```

Note: `no-multiple-empty-lines` removed from `rules` — same reason as backend. `eslintConfigPrettier` is the last entry so it wins over any formatting rules from previous configs.
